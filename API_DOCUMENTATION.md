# Smart Parking ML - API Documentation

This document describes the RESTful API endpoints available in the Smart Parking ML system. The backend is built with Flask and returns data in JSON format.

## Base URL
All endpoints are relative to the root URL: `http://127.0.0.1:5000/`

---

## 1. User & Prediction Endpoints

### 1.1. Get Current Status
Retrieves the real-time status of the parking lot, including capacity, current occupancy, slot statuses, and scheduled events.

- **URL:** `/api/status`
- **Method:** `GET`
- **Response Format:**
  ```json
  {
      "capacity": 100,
      "current_occupancy": 45,
      "slots": [true, false, true, ...], // Array of 100 booleans (true = occupied)
      "events": [
          {
              "name": "Concert",
              "start": "2026-05-15T20:00",
              "end": "2026-05-15T23:00",
              "extra_cars": 30
          }
      ]
  }
  ```

### 1.2. Get Occupancy Forecast
Returns an hourly occupancy forecast for the next 12 hours generated by the AI prediction engine.

- **URL:** `/api/forecast`
- **Method:** `GET`
- **Response Format:**
  ```json
  [
      {"time": "10:00", "occupancy": 45.2},
      {"time": "11:00", "occupancy": 52.8},
      ...
  ]
  ```

### 1.3. Smart Prediction
Makes an AI-driven prediction of the parking occupancy for a specific future date and time.

- **URL:** `/api/predict`
- **Method:** `POST`
- **Request Headers:** `Content-Type: application/json`
- **Request Body:**
  ```json
  {
      "datetime": "2026-05-16T14:30"
  }
  ```
- **Response Format:**
  ```json
  {
      "predicted_occupancy": 65.4,
      "occupancy_percentage": 65.4,
      "available_spaces": 34.6,
      "level": "Moderate",
      "reasons": ["Event Impact (Density: 80%)", "Normal traffic flow"],
      "performance_info": "Model MAE: 4.5%"
  }
  ```

## 2. Admin Management Endpoints

### 2.1. Update Quick Occupancy
Manually increments or decrements the total active vehicle count in the parking lot.

- **URL:** `/api/admin/occupancy`
- **Method:** `POST`
- **Request Headers:** `Content-Type: application/json`
- **Request Body:**
  ```json
  {
      "action": "inc" // Use "inc" to add a vehicle, "dec" to remove a vehicle
  }
  ```
- **Response:** Returns the updated status object (same as `/api/status`).

### 2.2. Toggle Slot Status
Reserves or frees up a specific parking slot on the live grid.

- **URL:** `/api/admin/slot`
- **Method:** `POST`
- **Request Headers:** `Content-Type: application/json`
- **Request Body:**
  ```json
  {
      "index": 5 // The 0-based index of the slot to toggle
  }
  ```
- **Response:** Returns the updated status object (same as `/api/status`).

### 2.3. Add New Event
Schedules a new event that will impact the AI prediction algorithm.

- **URL:** `/api/admin/event`
- **Method:** `POST`
- **Request Headers:** `Content-Type: application/json`
- **Request Body:**
  ```json
  {
      "name": "Local Match",
      "start": "2026-05-18T18:00",
      "end": "2026-05-18T21:00",
      "extra_cars": 40
  }
  ```
- **Response:** Returns the updated status object (same as `/api/status`).

### 2.4. Delete Event
Removes a scheduled event from the system using its array index.

- **URL:** `/api/admin/event/<index>`
- **Method:** `DELETE`
- **URL Parameters:** `index` (Integer) - The 0-based index of the event to delete.
- **Response:** Returns the updated status object (same as `/api/status`).
